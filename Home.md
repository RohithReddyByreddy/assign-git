### Working Process of SWIG:
SWIG uses an interface file with a .i extension, which serves as an extension of the C++ functions you create. This file is provided as input to SWIG. It is important that the interface file corresponds to the C++ code you write, as it contains various functions that are called from the SWIG interface file.
> Steps to Compile SWIG file and create Python extension :
* To execute these commands from the command line, you need to start by compiling the SWIG file. The command for compiling the SWIG file is as follows: swig -c++ -python <Interface_file_name>.i .
* When executed above command generates two files with extension .cxx and .py which are used for building python extension.
* One approach to building extension module for python is to compile with Distutills. Distutils takes care of making sure that your extension is built with all the correct flags, headers, etc. for the version of Python it is run with. In addition, distutils can handle installing your package into site-packages. A configuration file setup.py describes the extension.

        """
        setup.py file for SWIG example
        """

        from distutils.core import setup, Extension
        from distutils import sysconfig
        import numpy

        # Obtain the numpy include directory.  This logic works across numpy versions.
        try:
           numpy_include = numpy.get_include()
        except AttributeError:
           numpy_include = numpy.get_numpy_include()

         example_module = Extension('_arrayops',
            sources=['arrayops_wrap.cxx', 'arrayops.cpp'],
            include_dirs = [numpy_include],
         )
         setup (name = 'arrayops',
            version = '0.1',
            author = "SWIG Docs",
            description = """Simple swig example from docs""",
            ext_modules = [example_module],
            py_modules = ["arrayops"],
         ) 

* Once you have created the setup.py file, you need to execute the command python setup.py build_ext --inplace to build the extension module
* Now we can import swig python module in any python file and execute the functions.
* In this project, I have developed a batch file that automates the SWIG compilation and Python extension processes. When you click on the batch file, both the SWIG compilation and Python extension will be executed automatically.

### Inverse of a Matrix using Boost:
For inverse of a matrix there is a technique from boost library which uses LU factorization and back-substitution operations to compute the inverse matrix very rapidly.

    void matrix_inverse_func(double* arr4, double* arr7, int x4, int y4)
    {

    matrix<double> n(x4, y4);
    for (int i = 0; i < x4; ++i)
    {
        for (int j = 0; j < y4; ++j)
        {

            m(i, j) = arr4[index];
        }
    }
    typedef permutation_matrix<std::size_t> pmatrix;
    pmatrix pm(m.size1());
    int res = lu_factorize(m, pm);
    n.assign(identity_matrix<double>(m.size1()));
    lu_substitute(m, pm, n);

### Thread Building Blocks (TBB):
TBB is a C++ template library developed by Intel for parallel programming on multi-core processors. Using TBB, a computation is broken down into tasks that can run in parallel. The library manages and schedules threads to execute these tasks
    //addition function which executes on tbb for faster addition of arrays 

    double* add_func1(double* arr1, double* arr2, int x1, int y1, int z1)
    {
    long long end = (x1 * y1 * z1);
    tbb::parallel_for(tbb::blocked_range<long long>(0, end), [&](tbb::blocked_range<long long> r)
        {
            for (long long i = r.begin(); i < r.end(); i++)
            {
                arr1[i] = arr1[i] + arr2[i];
            } });
    return arr1;
    }
### OpenMP:
OpenMP is an API that supports direct multi-threaded, shared memory parallelism in C/C++ programs. OpenMP instructions are made in pragmas interpreted by the compiler.
    //Using openMP pragma technique for addition operation

    double* add_func2(double* arr1, double* arr2, int x1, int y1, int z1)
    {
    long long end = (x1 * y1 * z1);
    #pragma omp parallel for reduction(+ : arr1[i])
    for (int i = 0; i < end; i++)
    {
        arr1[i] = arr1[i] + arr2[i];
    }
    return arr1;
    }
### Memory mapping:
Memory mapping is a method that enables the contents of a file to be accessed in virtual memory. This mapping establishes a connection between a file and the memory space, allowing an application to directly read from and write to the file through memory operations. By utilizing managed code, you can access memory-mapped files similarly to how native Windows functions interact with them.
